Теория на сегодня:

1. Что будет на уроке. Веб-парсинг

Сегодня мы начинаем знакомиться с работой парсинга сайтов: рассмотрим, чем может пригодиться веб-парсинг и поговорим о его основных инструментах.

План занятия таков:

  • Мы узнаем, что такое парсинг;
  • Рассмотрим инструменты для веб-парсинга;
  • Разберём базовые понятия HTML и CSS.

✅ Веб-парсинг — это процесс извлечения данных из веб-страниц. Этот процесс может включать в себя анализ HTML-кода страницы для получения нужной информации, такой как тексты статей, информация о продуктах, списки электронной почты и многое другое.

Основная цель веб-парсинга в том, чтобы преобразовывать данные, имеющиеся в интернете на различных сайтах, в структурированную форму, которую потом будет легко анализировать и обрабатывать с помощью программ. Веб-парсинг близок к теме анализа данных — он помогает собирать данные для будущего анализа.

По сути, веб-парсинг похож на копирование информации с сайта, но не в ручном, а в автоматическом режиме. Программа как бы сканирует веб-страницу, находит нужную информацию и сохраняет её в удобном формате.

Веб-парсинг пригодится, если нам нужно, например, сохранить все статьи с сайта в несколько разных файлов (один файл — одна статья), или чтобы создать программу, которая будет сравнивать цены на один и тот же товар, продающийся на разных интернет-площадках.

Веб-парсинг достаточно популярен, поэтому с ним связано много заказов на фрилансе. После изучения этого модуля ты уже сможешь пробовать выполнять заказы — тем самым ты и пополнишь портфолио, и заработаешь.

Для чего нужен веб-парсинг:

  1. Автоматический сбор информации для анализа рынка;
  2. Мониторинг изменений цен на продукты для конкурентного анализа;
  3. Сбор данных для исследований;
  4. Автоматизация рутинных задач, таких как обновление данных на сайте.

Инструменты для веб-парсинга

Самый популярный инструмент для веб-парсинга — это HTTP-библиотеки.

✅ HTTP (HyperText Transfer Protocol) — это протокол передачи гипертекста. Это своего рода набор правил, по которым данные передаются между личным компьютером и сервером в интернете, на котором находится нужный нам сайт.

HTTP-запрос — это общение между компьютером и сервером.

Какими могут быть запросы? Например, при входе на сайт мы отправляем на сервер запрос для входа; сервер этот запрос обрабатывает и выдаёт какой-то ответ. При заполнении формы на сайте (например, формы регистрации) нажатием на кнопку Отправить мы тем самым отправляем запрос на сервер, и сервер, обработав этот запрос, возвращает ответ.


Одни из самых популярных инструментов для веб-парсинга — это:

  1. Requests — библиотека для выполнения HTTP-запросов в Python. Она отправляет HTTP-запросы на сервер.
  2. BeautifulSoup — библиотека, позволяющая легко извлекать информацию из HTML- и XML-файлов. Это модуль, позволяющий собирать информацию с сайтов. Благодаря этому модулю мы можем извлечь данные с сайта и собрать их в определённое место. BeautifulSoup может извлекать информацию из всех сайтов, написанных языком HTML.
  3. Scrapy — мощный фреймворк для веб-парсинга, предоставляющий всё необходимое для извлечения данных с веб-сайтов, их обработки и сохранения в структурированном формате.
  4. Selenium — программная библиотека для управления браузерами, которая используется для автоматизации браузеров. Этот инструмент позволяет полностью взаимодействовать с веб-страницей как пользователь, имитируя действия пользователя.

В целом существует огромное количество инструментов для веб-парсинга, мы здесь перечислили одни из самых популярных.


Как работают сайты?

Поговорим немного о том, как устроены сайты. Для более подробного изучения этой темы у нас будет отдельный модуль, а сейчас мы рассмотрим лишь базовые элементы разработки сайтов, чтобы в последующем понять принцип работы веб-парсинга.

Сайт состоит из двух частей: фронтенд и бэкенд.

✅ Фронтенд — это внешняя часть сайта, то, что видит пользователь, когда заходит на страницу: текст, изображения, кнопки, анимации.

✅ Бэкенд — это внутренняя часть сайта, где прописана вся логика работы сайта: логика регистрации, получения информации при заполнении форм, работа с базами данных и т.д.

Так как мы работаем с парсингом, нам нужно парсить информацию, находящуюся во фронтенде. Весь фронтенд использует три языка: HTML, CSS и JavaScript, но о JavaScript мы в рамках этого урока говорить не будем.

Любой сайт имеет определённую структуру и информацию. Вся эта структура и информация добавляется с помощью. языка HTML.

✅ HTML (HyperText Markup Language) — это язык разметки, используемый для создания веб-страниц.

С помощью HTML мы можем добавить на сайт любую информацию: текст, кнопки, аудио, видео, изображения, GIF и прочие элементы, воспринимаемые пользователем.

✅ CSS (Cascading Style Sheets) — язык, использующийся для оформления внешнего вида содержимого HTML.

Другими словами, CSS — это всё оформление сайта: цвета, анимации, шрифты, скругления краёв на изображениях и т.д. С помощью CSS сайту придаётся структурированный вид. Без CSS сейчас не делается ни одного сайта. Если сайт выглядит неудобно и некрасиво, в большинстве случаев мы не станем пользоваться таким ресурсом.

Разберёмся, как выглядит сайт и его код. Чтобы открыть консоль кода, можно воспользоваться либо мышью, либо горячими клавишами. Для примера рассмотрим сайт zerocoder.ru.

Чтобы открыть консоль с помощью клавиатуры, нужно нажать клавишу F12. Если F12 не срабатывает, нужно нажать сочетание Fn-F12.

С помощью мыши мы можем открыть код любого элемента: для этого надо выделить нужный элемент, щёлкнуть правой кнопкой мыши и выбрать в списке опцию Посмотреть код или Исследовать элемент (название этой функции может чуть отличаться).

Откроем консоль и рассмотрим код. При наведении мыши на какие-то части кода на сайте подсвечивается та часть сайта, которая относится к этому коду.

Таким образом мы можем пролистать весь код сайта и найти здесь нужную информацию. В нижней части консоли прописаны стили, относящиеся к тем или иным частям сайта.

В коде можно поменять какую-то информацию, но изменится она только на твоём компьютере и после перезагрузки страница вернётся к прежнему виду, так как снова подгрузится тот код, который лежит на сервере.

Как мы видим, код сайта Zerocoder довольно сложный; в следующем разделе мы рассмотрим более лёгкий код сайта и разберёмся, как он создаётся.

2. Первая страница на html

Поговорим о том, как создаются сайты, но коснёмся лишь базовых моментов их разработки, не погружаясь глубоко в эту тему.


1. Для написания кода создадим HTML-файл с названием index.html.

Первый файл в Python традиционно называют main (main.py); в разработке сайтов на HTML первому файлу обычно дают название index.

2. Находясь в PyCharm, правой кнопкой мыши кликаем по названию проекта, в списке выбираем New — File (или New — HTML File).

В случае, если мы выбрали File, в появившейся плашке "New file" нужно прописать: index.html и нажать клавишу Enter. Откроется пустой новый файл.
В случае, если мы выбрали в списке HTML-File, в новом файле уже будет прописан шаблон кода. Мы выбрали File, чтобы работать с изначально пустым файлом.

3. Чтобы в пустом файле появился базовый код, прописываем на первой строке !(восклицательный знак) и нажимаем клавишу Tab. В файле автоматически пропишется базовый код.

4. Обратимся к Нейрокоту, чтобы он прописал небольшой код для сайта. Заходим в чат Нейрокот и отправляем ему запрос: "Напиши код для создания простого сайта про котиков с небольшим оформлением с помощью CSS".

5. Копируем код, написанный Нейрокотом, и возвращаемся в файл index.html. Удаляем весь автоматически созданный код, так как Нейрокот уже всё прописал. Вставляем в файл скопированный код.

6. Чтобы открыть этот сайт в браузере, воспользуемся инструментарием PyCharm: в правой верхней части интерфейса PyCharm находим значки различных браузеров и выбираем нужный браузер, щёлкнув по нему. В браузере мы увидим версию сайта, созданную Нейрокотом.

Теги и их виды

Любой сайт состоит из тегов. Всё, что мы пишем в HTML — это теги. Теги — это составляющие части сайта, то есть кирпичики, из которых состоит сайт. Каждый тег добавляет на сайт определённые свойства и функционал, примерно так же, как функции в программировании, но теги обычно более просты в использовании.

Например, с помощью тега <img> мы можем добавить изображение на сайт; с помощью тега <p> добавляется текстовый элемент (параграф), и т.д. Всё, что мы видим на сайте, находится внутри тега <body>. Теги записываются в треугольных скобках.

Теги бывают двух видов: одинарные и парные. Одинарные теги пишутся с помощью открывающей треугольной скобки вначале и закрывающей треугольной скобки в конце.

Парные теги имеют начало в виде открывающей и закрывающей треугольной скобки, и конец, который обозначается такими же скобками, но с использованием слэша. Всё, что добавляется на сайт, прописывается между двумя тегами — открывающим и закрывающим.

Мы можем заменить любой текст в коде: например, вместо "Привет, котики!" напишем "Мур мяу", после чего снова воспользуемся кнопкой браузера в правом углу и увидим соответствующие изменения на веб-странице.


Классы и идентификаторы (id) для тегов

При использовании парсеров мы будем извлекать информацию из тегов. Здесь есть некоторые особенности. Названия тегов могут пересекаться, или быть похожими, или на сайте может быть использовано несколько одинаковых тегов, но для разной информации. Например, для карточек с товарами может использоваться один и тот же тег, что и для карточек с отзывами, а нам нужна информация только о товарах.


Как обозначить это при парсинге?

В HTML, помимо тегов, существуют классы и идентификаторы (id). Они позволяют объединять теги в группы, например, для создания группы карточек товаров. Карточки товаров, как правило, используют тег, работающий с текстом, но чтобы выделить карточки среди прочего текста на сайте, мы присвоив им класс, например, "card". Классы можно создавать с любыми названиями.


Идентификаторы (id) так же, как и классы, помогают выделять определённые теги, но существует важное отличие: id — это уникальный идентификатор, который присваивается только одному тегу и больше не может быть использован ни для какого другого тега. Класс же можно присваивать нескольким тегам.


Рассмотрим это на примере.

  1. Используем тег <p> (этот тег "параграф", который добавляет на сайт любой текст). Прописываем в файле HTML строчку кода: <p>Это мой первый сайт!</p>. Далее копируем и несколько раз на новых строках кода вставляем прописанный нами текст: <p>Это мой первый сайт!</p>

2. А теперь представим, что нам надо присвоить нескольким из этих одинаковых тегов какой-то класс. Для этого прописываем в открывающий тег класс: <p class="first">Это мой первый сайт!</p>. Зададим этот же класс тегу на следующей строке.


3. Присвоим другому тегу id: для этого прописываем в открывающем теге id: <p id="test">Это мой первый сайт!</p>

Если ты пропишешь <p id="test"> для любого второго тега, программа подчеркнёт этот элемент как ошибку, так как id можно присваивать только одному тегу.

На сайте содержится огромное количество информации, созданной с использованием тега <p>: статьи, информация об авторе и многое другое. Если при парсинге в параметрах поиска мы зададим только сам тег <p>, мы получим много лишней информации. Поэтому в процессе парсинга мы ищем информацию, указывая как тег (<p>), так и класс (например, text), чтобы отфильтровать только нужные нам данные.

Для примера вернёмся на сайт Zerocoder. Выделяем какой-нибудь заголовок и правой кнопкой мыши выбираем просмотр кода. В коде подсвечивается тег <h2> — это тег заголовка, у которого есть свой класс (в данном случае tn-atom).

Таким образом мы можем просматривать информацию в коде и, выбрав то, что нам нужно, использовать класс этой информации для парсинга. Обычно для однотипной информации на сайте тегам присваивается один и тот же класс, так как для этих классов задаются стили. Поскольку стили у одинаковых элементов обычно одинаковые, это обеспечивает единообразие оформления.

3. Базовые html теги

Рассмотрим ещё несколько тегов, которые могут пригодиться и в дальнейшей работе с HTML, и для выполнения домашнего задания.

Для начала рассмотрим тег <body>. В этом теге содержится вся информация, доступная на сайте. Для отображения текста на сайте можно просто добавить его внутрь тега <body>, без использования дополнительных тегов.


Все теги, находящиеся вне тега <body> — это различные элементы и плагины, подключенные к сайту, где обычно не прописывается содержимое сайта. Таким образом, вся информация, с которой мы будем работать, размещается внутри тега <body>, при этом расстояние между открывающим и закрывающим тегами может быть практически любым.


Тег параграфа и теги заголовков

Как мы уже говорили, тег <p> — это тег параграфа, в котором прописывается любой текст на сайте. Тег <p> всегда начинается с новой строки.


1. Для примера с новой строки прописываем внутри тега <body> обычный текст: <p>Просто текст параграфа на сайте</p>


2. Перед этим текстом прописываем заголовок: <h1>Заголовок 1 уровня</h1>.

Существуют шесть уровней тегов заголовков. Это заголовок <h1>, <h2>, <h3>, <h4>, <h5> и <h6>.

3. Прописываем на следующих строках заголовки следующих уровней (можно просто скопировать код из пункта 2, а затем заменить последовательно цифры в тексте и названия тегов:

<h2>Заголовок 2 уровня</h2>

<h3>Заголовок 3 уровня</h3> 

<h4>Заголовок 4 уровня</h4> 

<h5>Заголовок 5 уровня</h5> 

<h6>Заголовок 6 уровня</h6>

Следи за тем, чтобы закрывающий тег был таким же, как и открывающий, но с добавлением / (слэш). В PyCharm при изменении открывающего тега закрывающий тег заменяется автоматически, но в любом случае за этим нужно следить.

4. Запускаем прописанный код с помощью кнопки браузера в правом верхнем углу PyCharm и видим, что мы с помощью этих тегов создали заголовки разных уровней, то есть, разных размеров. Под заголовками мы видим обычный текст, созданный с помощью тега <p>.

Теги разделителя и отступов

1. С помощью одинарного тега <hr> можно добавить на сайт разделитель. Прописываем: <hr> и в результате получаем линию, разделяющую части сайта.


2. С помощью тега <br> добавляются отступы: например, чтобы во фразе "Просто текст параграфа на сайте" часть текста перенести на новую строку, внутри уже существующего текстового тега пропишем тег <br>. Получится строка: <p>Просто текст параграфа<br>на сайте</p>. В результате на сайте часть текста перенесётся на следующую строку.

Теги внутри тегов прописывать можно: мы пишем весь сайт внутри тега <body>, и точно так же внутри каждого тега можно добавлять новые теги.

3. Чтобы увеличить расстояние между строчками в тексте, нужно просто добавить ещё один тег <br>: <p>Просто текст параграфа <br><br> на сайте</p>.

Тег для добавления изображений

Мы уже знаем, что для тегов существуют классы и id. Всё, что мы дописываем внутри треугольных скобок для тега, называется атрибутами.

✅ Атрибуты в HTML — это дополнительные сведения или инструкции, которые могут быть добавлены к тегам для определения их свойств или поведения. Атрибуты обычно указываются внутри открывающего тега.

Классы и id — это необязательные атрибуты. Наряду с ними существуют и обязательные атрибуты, без которых некоторые теги работать не будут.

Чтобы добавить на сайт изображение, используется одинарный тег <img>. Но если мы просто напишем <img>, изображение на сайте не появится. Нужно указать в коде конкретное изображение. Для этого и существуют атрибуты. Для тега <img> прописываем обязательный атрибут src. В результате строчка кода будет выглядеть так: <img src="Путь к изображению">.

Путь к изображению можно прописать двумя способами: ссылкой на изображение в интернете либо на существующее на компьютере изображение. Разберём оба варианта.

Скачиваем изображение на свой компьютер и перетаскиваем его в проект PyCharm.

Внутри тега <img src=""> в кавычках нужно прописать путь к изображению. Если изображение находится в какой-то папке, нужно прописывать путь с учётом папки. Так как в нашем случае изображение находится не в папке, достаточно просто прописать название этого изображения. В результате строчка кода будет выглядеть так: <img src="images.jpeg">. При новом запуске мы увидим это фото на сайте.

Многие предпочитают добавлять изображение с помощью ссылки, чтобы не скачивать его на компьютер. Для этого находим в поисковике нужное изображение, щёлкаем по нему и правой кнопкой мыши выбираем опцию Копировать URL картинки или Скопировать ссылку на изображение. После этого, вернувшись на страницу кода, в строке <img src="ссылка"> вставляем скопированную ссылку.

Ссылка может быть огромной: такие ссылки лучше не брать и поискать другое изображение. Также фото по ссылке может быть бóльшего размера, чем нам нужно; размер фото тоже можно регулировать с помощью атрибутов, но сейчас мы не будем уделять этому внимание.

Тег для добавления ссылок

Мы разобрались, как с помощью кода добавлять на сайт заголовки, разделительные полосы, обычный текст и изображения. Теперь поговорим о том, как добавлять на сайт ссылки. Для этого понадобится тег <a> с обязательным атрибутом.

На новой строке кода пропишем: <a>Вот тут можно купить курс по программированию на Python</a>. Но без атрибута эта ссылка будет выглядеть на сайте как обычный текст и не будет кликабельной, потому что мы не назначили никакой ссылки. Чтобы здесь появилась какая-то ссылка, нужно прописать обязательный атрибут href. Скопируем нужную ссылку и вставим её в код, например: <a href="zerocoder.ru">Вот тут можно купить курс по программированию на Python</a>. Открыв сайт, мы увидим, что теперь ссылка кликабельна и ведёт на сайт Zerocoder.

Сегодня мы рассмотрели самый базовый набор тегов, необходимый для создания простого сайта с текстом, изображениями и ссылками. Мы также можем добавить сюда кнопки, но у них не будет функционала, так как кнопки нужно программировать с помощью JavaScript. Для добавления кнопки нужно использовать тег <button> и прописать, например: <button>Купить курс</button>. Теперь на сайт добавилась кнопка.

Коротко поговорим об ещё одном теге: <div>. С помощью этого тега сайт делят на разделы (блоки): блок меню, блок с описанием сайта, блок с каталогом, блок отзывов и т.д. Это деление также можно отображать с помощью стилей, но их мы сейчас затрагивать не будем.

Зайдя опять на сайт Zerocoder в консоль кода, мы можем убедиться в том, что сайт свёрстан с помощью тегов <div>. Большое количество сайтов создаются с помощью этого тега.
